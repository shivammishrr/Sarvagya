🏛 High-Level Architecture

🔄 Agent Loop

Synchronous think-tool-observe loop. One action per iteration — simple, observable, debuggable.

Key Design Decisions
  • One action per iteration — single tool call, observe result, repeat. No multi-tool turns.
  • Sliding window context — truncate_messages() keeps last 20 turns to manage token limits.
  • Tool result truncation — capped at 10,000 characters per result.
  • Memory index — prepended to system prompt as markdown list with 80-character previews.
  • Exception safety — LLM calls wrapped in try/except, returns error AgentResult on failure.
  • 50 iteration cap — hard limit prevents infinite loops.

🔌 Adapter Details

OpenAIAdapter

Works with OpenAI and Groq (OpenAI-compatible).

  • SDK: openai>=1.0.0
  • Tool schema: additionalProperties: false
  • Messages: native dict format with tool_calls array
  • ToolCall: arguments as JSON string → ToolCall.arguments
  • Response: choice.message.content or ""

AnthropicAdapter

Anthropic Claude via anthropic SDK.

  • SDK: anthropic>=0.30.0
  • Tool schema: input_schema format (no additionalProperties)
  • Messages: content blocks — tool_use + tool_result
  • ToolCall: dict(b.input) → ToolCall.arguments as dict
  • System: separate system param, not in messages
  • Max tokens: fixed max_tokens=4096

LocalSandbox

Local subprocess execution.

  • Mechanism: subprocess.run(shell=True)
  • Workdir: provided path or tempfile.mkdtemp()
  • Timeout: ms → seconds conversion, default 120s
  • Output: stdout + stderr combined

FileMemory

Markdown files with YAML frontmatter.

  • Format: {key}.md with ---\nkey: value\n--- frontmatter
  • Index: MEMORY.md with links to all entries
  • save(): writes file + appends to index
  • load(): parses frontmatter + content

TavilySearch

Web search via Tavily API.

  • SDK: tavily-python>=0.7.0
  • Results: markdown formatted (title, URL, content)
  • Conditional: only registered if TAVILY_API_KEY is set
